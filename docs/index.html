Getting started

Installation

via package managers

yarn add vue-next-select
npm i vue-next-select
import { createApp, ref } from 'vue'
import VueSelect from 'vue-next-select'
import 'vue-next-select/dist/index.min.css'

const app = createApp({
  name: 'app',
  setup () {
    const model = ref(null)
    const options = ['vue', 'next', 'select']
    return {
      model,
      options
    }
  },
  template: `
    <vue-select
      v-model="model"
      :options="options"
    ></vue-select>
  `
})

app.component('vue-select', VueSelect)
app.mount('#app')

via CDN

<script src="https://unpkg.com/vue-next-select"></script>
<link rel="stylesheet" href="https://unpkg.com/vue-next-select/dist/index.min.css" />
app.component('vue-select', VueNextSelect)

Options

vue-next-select accepts arrays of any type values to use as options:

<vue-select :options="['Select', 'Options']"></vue-select>

By default, the label of option will be the option itself. When your option is an object or non-primitive values, it would be more make sense to use a string value to display it, so you have to provide an additional label-by: string | Function: 

<vue-select
  label-by="country"
  :options="[{ country: 'United States', code: 'USA' }, { country: 'Taiwan', code: 'TWN' }]"
></vue-select>

v-model

The most common use case for vue-next-select is to have the chosen value synced with a parent component. 

<vue-select v-model="selected" :options="['Vue.js', 'JavaScript']"></vue-select>

When works with non-primitive options, there will be situations where maybe you just need to return a value from an object instead of the object itself. You could use value-by: string | Function to do this: 

<!-- The selected will be 'Vue.js' or 'Laravel' here -->
<vue-select
  v-model="selected"
  value-by="name"
  :options="[{ name: 'Vue.js', language: 'JavaScript' }, { name: 'Laravel', language: 'PHP' }]"
></vue-select>

vue-next-select use null as empty value by default for single mode select. You could use empty-model-value: any to override this behavior: 

<vue-select
  v-model="selected"
  :empty-model-value="undefined"
  :options="[{ name: 'Vue.js', language: 'JavaScript' }, { name: 'Laravel', language: 'PHP' }]"
></vue-select>

Single select

Select with search

To make it be searchable / filterable, you just need to set searchable: boolean to true.

By default, existing options will be filtered by the search query (case insensitive).

If you need to customize this behavior, just handle @search:input or @search:change events and apply an additional visible-option prop to dynamically change rendered options. For more details, see asynchronous-select example below.

Multiple select

By default, vue-next-select supports select a single value. If you need multiple values, use the multiple: boolean prop, when the multiple is true, v-model / modelValue must be an array.

If you need to limit the number of selections, use min: number and/or max: number.

When close-on-select: boolean is true, it will hide the dropdown after select or remove an option.

Tagging

Group

You could define an option as group by adding a group (default, set it via group-by prop) to true.

It uses the label-by to locate the group label as same as a normal option, and the value-by to destinguish which option belongs to current group.

When selecting a group option it will select all the options belong to it, and deselect all if those options are already selected.

Despite that the options are grouped, the options must be a flat array of objects.
